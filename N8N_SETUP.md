# n8n Setup — Internship Outreach Pipeline

## Prerequisites

- [n8n](https://n8n.io/) installed locally (`npx n8n` or Docker)
- SMTP credentials configured in n8n (for Send Email node)
- Python venv with dependencies at `C:/Users/USER/Documents/Internship agent/venv/`

## Import the Workflow

1. Open n8n at `http://localhost:5678`
2. Go to **Workflows → Import from File**
3. Select `n8n_workflow.json`
4. In node **Send Email: Outreach**, select your SMTP credential (or create one)

## Configure SMTP in n8n

1. Go to **Settings → Credentials → Add Credential → SMTP**
2. For Gmail SMTP use:

- Host: `smtp.gmail.com`
- Port: `587`
- User: your full Gmail address
- Password: Gmail App Password
- Secure: off (STARTTLS on 587)

3. Test the connection and save

## Optional: Dynamic sender address

- The workflow uses `fromEmail = {{$env.N8N_SMTP_FROM || 'David Ihegaranya <dihegaranya@gmail.com>'}}`
- To change sender globally without editing the workflow, set env var `N8N_SMTP_FROM`
- Example: `N8N_SMTP_FROM="David Ihegaranya <dihegaranya@gmail.com>"`

## How It Works

The workflow has **two independent flows**:

### Flow 1: Research (Manual Trigger)

```
[Click Execute] → Get Next Company → Run Python Pipeline → Read Prompt → Mark as Researched
```

- Click **"Execute Workflow"** to research the next pending company
- Python pipeline runs DuckDuckGo search + Hunter.io email finding/verification
- The Gemini prompt is saved to `drafts/{company}_prompt.txt`
- You open the prompt file → paste into Gemini Pro → copy Gemini's reply → paste into `drafts/{company}_drafts.txt`

### Flow 2: Send (Scheduled, every 30 min)

```
[Every 30 min] → Check Drafts Folder → Parse EMAIL Blocks → Validate → Send via SMTP → Mark Done
```

- Runs automatically every 30 minutes
- Processes one researched company per schedule cycle for safety
- Checks all `drafts/{company}_drafts.txt` files for `---EMAIL---` blocks
- Pre-filters duplicate recipients (already in `send_log.json`) before sending
- Parses and sends each email via SMTP Send Email node (30s delay between, capped by daily budget)
- Updates `progress.json` to mark the company as completed
- No self-notification emails (keeps your inbox clean)

## Your Workflow (from n8n)

1. **Click "Execute Workflow"** in n8n → researches next company → saves prompt to file
2. **Open `drafts/{company}_prompt.txt`** → copy contents → paste into **Gemini Pro** (gemini.google.com)
3. **Copy Gemini's reply** → paste into `drafts/{company}_drafts.txt` → save
4. **Wait** (or trigger manually) → n8n auto-detects drafts → sends emails
5. **Repeat** → click Execute again for the next company

## Check Progress

From terminal:

```bash
python pipeline.py status
```

Or check `progress.json` directly.

## Commands (still available from CLI)

| Command                                          | Description                                                 |
| ------------------------------------------------ | ----------------------------------------------------------- |
| `python pipeline.py status`                      | View progress                                               |
| `python pipeline.py research -c PiggyVest`       | Research one company manually                               |
| `python pipeline.py send -c PiggyVest --dry-run` | Preview emails                                              |
| `python pipeline.py send -c PiggyVest`           | Send emails manually                                        |
| `python pipeline.py loop`                        | Interactive loop (research → prompt → wait → send → repeat) |
| `python pipeline.py reset -c PiggyVest`          | Reset a company                                             |
| `python pipeline.py reset --all`                 | Reset all progress                                          |

## File Structure

```
drafts/
  piggyvest_prompt.txt    ← Gemini prompt (auto-generated by research)
  piggyvest_drafts.txt    ← YOU paste Gemini reply here
  piggyvest_data.json     ← Prospect data backup
progress.json             ← Tracks completed/researched companies
```

## Notes

- Hunter.io free tier: 25 email-finder + 50 email-verifier calls/month
- Gmail: 30s delay between sends, hard cap `18/day` to reduce spam risk
- All emails include an opt-out line at the bottom
- `progress.json` is shared between n8n and CLI — use either interchangeably
- `send_log.json` prevents duplicate outreach to the same recipient across runs
